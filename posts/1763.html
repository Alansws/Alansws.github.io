我与 AI 助手：从零到一搭建个人博客的完整记录

前言：一直想拥有一个属于自己的博客，用来记录技术和生活。作为一名开发者，我决定使用 Hexo 静态博客框架，并借助 AI 助手 Cursor 来完成整个搭建过程。这篇文章，便是我与 AI “拉扯”与协作，从环境配置到深度主题定制的全程实录。

阶段一：启程与第一个“下马威”

User
我打算使用 Hexo 框架和 NexT 主题制作一个个人博客网站，最终目标是能通过 Git 一键部署到 GitHub Pages。请你根据我的计划，进行一个完整的项目规划。

Cursor
好的，以下是根据你的需求制定的完整项目规划，涵盖了从技术选型、开发流程到注意事项的全部内容。

个人博客网站开发完整规划

  1. 项目整体架构
    • 技术选型: Hexo, NexT 主题 (后更换为 Anzhiyu), Node.js, Git, GitHub Pages, Giscus 评论系统。
    • 主要功能: 文章发布、主题美化、本地搜索、评论系统、一键部署。
  2. 开发关键节点
    • 环境初始化 -> 主题安装 -> 核心配置 -> 内容创建 -> 功能增强 -> 部署配置 -> 上线。
  3. 注意事项
    • 所有占位符（如 GitHub 用户名）需替换为个人实际信息。
    • GitHub 仓库需提前创建，格式为 your-username.github.io。
    • 建议使用 Node.js 的 LTS 版本。

确认规划后，我们即可开始第一步。

阶段三：部署与认证的“世纪难题”

User
所有配置都完成了，现在准备部署。

Cursor
收到，现在我们进入部署环节。首先安装 Git 部署插件，然后在 _config.yml 中配置好你的 GitHub 仓库地址。

User
运行 hexo deploy 命令后提示报错，请你分析并提出解决方案。

Cursor
这个报错的原因是，GitHub 在 2021 年后就不再支持使用账户密码来推送代码了，必须使用 Personal Access Token (个人访问令牌) 或 SSH key 进行身份验证。

推荐使用 SSH Key，更安全也更方便。

User
我 GitHub 上之前好像申请过一个 SSH Key，此时需要重新 New 一个吗？以及我需要执行“生成 SSH Key”这一步吗？我找不到 ~/.ssh/id_ed25519.pub 这个文件。

Cursor
你的问题很细致，我来一步步教你。

  1. 是否需要新建？
    如果本地已经有 SSH Key，不需要新建。我们先来检查一下。

  2. 检查本地是否已有 SSH Key
    在终端输入：ls ~/.ssh。如果输出中有 id_ed25519.pub 或 id_rsa.pub，说明你已经有了。

  3. 复制公钥内容
    用 cat ~/.ssh/id_rsa.pub 命令查看并复制公钥内容。

  4. 添加公钥到 GitHub
    将复制的内容粘贴到 GitHub 的 Settings -> SSH and GPG keys 页面。

  5. 修改 Hexo 部署配置为 SSH 格式
    打开 _config.yml，将 repo 字段改为 SSH 格式：

  6. 测试并重新部署

    • 测试连接：ssh -T git@github.com
    • 重新部署：hexo clean && hexo generate && hexo deploy

User
我本地终端运行 cat ~/.ssh/id_rsa.pub 命令后是有输出的，但是我添加 SSH Key 到 GitHub 并保存时，提示 Key is already in use，我该怎么做？

Cursor
这个提示说明你的这把 SSH 公钥已经被添加到你的 GitHub 账号了。

你现在该这么做：

  1. 无需重复添加，直接用它进行认证即可。
  2. 检查 deploy 配置，确保 repo 字段是 SSH 格式 git@github.com:...。
  3. 测试 SSH 连接，运行 ssh -T git@github.com，如果返回 Hi your-username! You've successfully authenticated... 就说明配置完全没问题。
  4. 重新部署，运行 hexo clean && hexo generate && hexo deploy。

User
提示我“Congratulations, you’re all set!”，似乎已经成功了。

阶段五：主题深度定制的“分类”拉锯战

User
目前博客网站一进入就是一篇文章，没有首页吗？以及我想修改主题该怎么样让它更专业更好看呢？

Cursor
Hexo 默认首页就是文章列表页，你只看到一篇是因为你只写了一篇。多写几篇就会变成列表了。
关于主题美化，你正在使用的 Anzhiyu 主题支持高度自定义。我们可以从分类显示逻辑开始优化。

User
我设置了二级分类，比如文章的 front-matter 里写 categories: [大学, 大学生涯]，但是 category-bar（分类栏）上同时显示了“大学”和“大学生涯”，这样子感觉更怪了，怎么办？

Cursor
你遇到的问题是 Hexo 会将每一级分类都单独作为一个节点显示。我们可以通过修改主题模板，让分类栏只显示一级分类。

  1. 定位模板文件：分类栏的渲染逻辑在 themes/anzhiyu/scripts/helpers/catalog_list.js。
  2. 修改渲染逻辑：我们找到 catalog_list 这个 helper 函数，在 map 函数前加上 filter，只筛选出没有父级的分类（即一级分类）。 这样，分类栏就只会显示“大学”这个一级分类了。

User
这样修改后，点击 category-bar 的“大学”进入的是“分类-大学”页面，但点击我自己在首页顶部配置的“大学”按钮，进入的还是“分类-大学生涯”页面。我感觉逻辑有问题。

Cursor
你的感觉是对的，原因是：同一个“大学”按钮，category-bar 和首页顶部按钮跳转到的不是同一个页面，用户体验不一致。

推荐做法：统一它们的 path，都指向一级分类页面 /categories/大学/。
修改 _config.anzhiyu.yml 文件中，首页顶部按钮的 path：

  1. 添加自定义 CSS 样式：
    为了让二级分类列表更美观，可以在 themes/anzhiyu/source/css/_extra/fix/ 目录下新建一个 CSS 文件（主题会自动加载），并添加一些美化样式。

User
如图，这是进入“大学”后的画面，但我刚才不是让你在这个页面再加上筛选 sub-categories 分类的按钮吗，为什么目前没有看到？

Cursor
你的截图显示了 Pug 代码没有被正确渲染。这通常是 Pug 对缩进极其敏感导致的，或者是数据获取方式有细微差别。

解决办法：
请严格检查 category.pug 中插入代码的缩进，并确保使用 site.categories.data 来获取分类数据。

同时，为了调试，你可以在页面上临时输出变量，看看是否成功获取到了分类和子分类的数量。

通过这样的调试和修正，最终二级分类筛选区域就能成功显示出来了。
